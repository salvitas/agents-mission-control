# TRADING BUILDOUT v1

## Objective
Build a robust market decision system across:
- Macro
- Metals
- Equities
- Crypto (primary focus)

Goal: Produce high-quality trade plans quickly, then execute only with explicit user confirmation.

---

## 1) Top-Down Framework

### A. Regime Detection (Daily)
Classify environment first:
- Risk-on / Risk-off / Neutral
- Growth vs Inflation impulse
- Liquidity expanding vs contracting

Core proxies to monitor:
- DXY
- US10Y / real yields
- VIX
- BTC dominance, ETH/BTC
- Gold, Silver, Copper
- SPX, NDX, Russell

### B. Theme Mapping (Daily/Weekly)
Map flows into themes:
- AI / growth beta
- Energy / commodities
- Precious metals
- L1 / L2 / memecoins / DeFi / AI crypto

### C. Asset Selection
Pick assets with:
- Clear trend structure
- Sufficient liquidity
- Catalyst alignment
- Good R:R setup

---

## 2) Playbook Library (Initial)

### Playbook 1: Pullback in Uptrend
- Use when higher timeframe trend is intact
- Enter near support/retest
- Invalidate below structure break

### Playbook 2: Breakout + Retest
- Use after compression/range break
- Require volume confirmation
- Enter on hold/retest, not first spike

### Playbook 3: Event-Driven Momentum
- Use for earnings/FOMC/CPI/ETF/major listings
- Smaller size due to volatility
- Predefine hard invalidation

### Playbook 4: Mean Reversion (Countertrend)
- Use only in range-bound regimes
- Tight risk and quicker profit taking

---

## 3) Risk Engine v1

Hard rules:
- Risk per trade: 0.5–1.0% max
- Daily max loss: 2.0%
- Weekly max drawdown: 5.0%
- No averaging losers without a predefined ladder plan
- Stop must exist before entry
- If market regime unclear: cut size by 50%

Execution safety:
- Never place live orders without explicit user confirmation
- Confirm: symbol, side, size, entry, stop, TP(s), leverage
